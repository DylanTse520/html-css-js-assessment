Positioning options

This is relative positioning. It takes its place in the normal flow, but you can modify its final position even to overlap with other elements. This position is useful when you have a position: absolute child and want that to stick to its parent.
This is absolute positioning. It does not exists in the normal document flow. You can create isolated UI features that don't interfere with the layout of other elements on the page. For example, it is useful when you want to create a tooltip.
Top
This is fixed positioning. It no longer exists in the normal document flow just like absolute positioning. But the difference is that it is located relative to the viewport. This is useful to create things that stays on the webpage no matter where you scroll. For example, a scroll to top element that scrolls the page to the top when you click.
This is sticky positioning. It is basically a hybrid between relative and fixed position. It allows a positioned element to act like it's relatively positioned until it's scrolled to a certain threshold, after which it becomes fixed. It is useful to create navbar which scrolls with the page until a certain point and then stick to the top of the page.
